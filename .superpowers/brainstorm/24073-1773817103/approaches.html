Past Recipes Page — Two Approaches

How should we build the completion flow + past recipes feature?

A

New Standalone Page — /past-recipes

Create a brand new PastRecipesPage component at a new route. After finishing cooking, navigate to /past-recipes with a "justFinished" flag + the recipe name in route state. The page shows a congrats banner at the top when coming from cooking, then lists all past recipes below. MainPage gets a "View Past Recipes" button that navigates to the same route (without congrats).

Pros

  • Clean separation — one page, one purpose
  • Reusable from both CookingPage and MainPage
  • Simple routing — just add one route

Cons

  • Similar to existing RecipeLibraryPage (some overlap)
B

Extend RecipeLibraryPage with Completion Mode

Reuse the existing /recipes RecipeLibraryPage. Add a "justFinished" state flag — when present, show a congrats banner at the top and the "Back to Home" button at the bottom. When accessed normally (from MainPage button), show it without the congrats. Rename the page title to "Past Recipes".

Pros

  • No new files — extends existing page
  • No duplicate recipe-listing logic

Cons

  • Mixes two concerns in one component (completion flow + library)
  • RecipeLibraryPage has delete functionality — may not want that in the completion flow

I recommend Option A — a clean new page keeps the completion flow focused and avoids mixing delete buttons into the "congrats" experience.
